# PRD: 포토 다이어리 앱 (육아 일기)
**문서 버전:** v1.0  
**작성일:** 2026-03-20  
**대상:** MVP 개발팀 인수인계용

---

## 1. 문제 정의

### 배경
육아 중인 부모는 아이의 성장 순간을 수백~수천 장의 사진으로 기록하지만, 사진이 앨범에 무질서하게 쌓일 뿐 **의미 있는 기억으로 정리되지 못한다.**

### 핵심 문제
- 사진은 많지만 날짜별·이벤트별로 정리하는 데 시간이 너무 많이 걸림
- 일기 앱과 사진 앱이 분리되어 있어 함께 관리하기 불편함
- 바쁜 육아 일상 속에서 일기를 "새로 작성"하는 심리적 허들이 높음
- 소중한 기억을 특정 가족/지인과만 선택적으로 공유할 수단이 없음

### 해결 방향
> 사진을 먼저 보여주고 → 일기를 쓰게 만든다.
> 타임라인이 일기의 뼈대가 되어, 부모가 사진을 고르기만 해도 일기가 완성되는 경험을 제공한다.

## 2. 사용자 스토리

| # | 역할 | 원하는 것 | 이유 |
|---|------|-----------|------|
| US-01 | 육아 중인 부모 | 날짜별 타임라인에서 오늘 찍은 사진을 한눈에 보고 싶다 | 어떤 사진을 찍었는지 맥락 파악이 먼저 필요하기 때문 |
| US-02 | 육아 중인 부모 | 사진을 선택하면 바로 일기 작성 화면으로 이동하고 싶다 | 빈 화면에서 일기를 시작하는 것은 부담스럽기 때문 |
| US-03 | 육아 중인 부모 | 작성한 일기를 날짜 순서로 다시 볼 수 있어야 한다 | 아이의 성장 스토리를 시간 순으로 회고하고 싶기 때문 |
| US-04 | 육아 중인 부모 | 특정 일기를 조부모나 배우자에게만 공유하고 싶다 | 전체 공개는 싫지만 가까운 가족에게는 보여주고 싶기 때문 |
| US-05 | 육아 중인 부모 | 앱을 켜지 않아도 기록을 남기도록 알림을 받고 싶다 | 바쁜 육아 중에는 앱을 잊기 쉽기 때문 |

## 3. 핵심 기능 (상세 요구사항)

### F-01. 사진 타임라인 뷰 ⭐ 최우선
> 앱의 메인 화면. 기기 사진첩을 날짜 기준으로 자동 그룹핑하여 보여준다.

- **사진첩 접근 권한** 요청 → 허용 시 기기 갤러리 사진을 불러옴
- 날짜별(Day) 섹션 헤더로 그룹핑 (예: `2026년 3월 20일 목요일`)
- 섹션 내 사진은 촬영 시각 오름차순 정렬
- 썸네일 그리드(3열) 형태로 표시
- 이미 일기가 작성된 날짜는 섹션 헤더에 **"일기 있음"** 배지 표시
- 월(Month) 단위 스크롤 점프 가능한 사이드 인덱스 또는 상단 월 필터 제공
- 사진 미접근 상태에서도 앱 진입 가능 (텍스트 전용 일기 작성 허용)

---

### F-02. 사진 선택 → 일기 작성
> 사진을 선택하는 행위가 일기 작성의 시작점이 된다.

- 타임라인에서 사진 탭 → **선택 모드** 진입 (체크박스 UI)
- 동일 날짜 내 다중 선택 가능 (최대 10장/일기)
- 다른 날짜 사진 동시 선택 불가 (날짜 경계 명시 안내)
- 선택 완료 후 **"일기 쓰기"** CTA 버튼 → 일기 작성 화면으로 이동
- 작성 화면 구성:
  - 상단: 선택한 사진 가로 스크롤 프리뷰 (순서 드래그 변경 가능)
  - 날짜 자동 입력 (수동 수정 가능)
  - 제목 입력 (선택, 최대 50자)
  - 본문 텍스트 에디터 (최대 2,000자, 줄바꿈 지원)
  - 감정 이모지 태그 선택 (기쁨/사랑/뿌듯함/피곤/슬픔 등 10종)
  - 저장 / 임시저장 / 취소 버튼

---

### F-03. 일기 목록 및 상세 뷰
> 작성된 일기를 시간 순으로 탐색한다.

- **목록 뷰:** 날짜 내림차순 정렬, 카드형 UI (대표 사진 1장 + 제목 + 본문 2줄 미리보기 + 날짜)
- **상세 뷰:** 사진 풀스크린 스와이프 + 본문 텍스트 + 감정 태그 표시
- 일기 **수정** 및 **삭제** 기능 (삭제 시 확인 모달)
- 월별 캘린더 뷰: 일기가 있는 날짜에 점(dot) 표시, 탭 시 해당 일기로 이동

---

### F-04. 선택적 공유
> 특정 일기를 지정한 사람에게만 공유한다.

- 일기 상세 화면에서 **"공유"** 버튼 제공
- 공유 방식: **비공개 링크 생성** (링크를 아는 사람만 열람 가능)
- 링크 유효기간 설정: 1일 / 7일 / 30일 / 무제한
- 링크 수신자는 **읽기 전용** 웹 뷰로 열람 (앱 설치 불필요)
- 링크 **비활성화(취소)** 기능 제공
- 공유된 일기는 목록에서 **공유 아이콘** 표시

---

### F-05. 알림 (기록 독려)
> 사용자가 기록 습관을 유지하도록 돕는다.

- 사용자가 직접 설정하는 **일일 리마인더** (예: 매일 오후 9시)
- 알림 문구: "오늘 [N]장의 사진이 쌓였어요. 기록을 남겨볼까요?"
  - N = 오늘 날짜 기준 기기 사진 수
- 알림 ON/OFF 및 시간 변경은 설정 화면에서 관리
- 일기를 이미 작성한 날은 알림 발송 안 함

---

### F-06. 계정 및 데이터 관리
> 멀티 디바이스 접근과 데이터 보호를 위한 최소 인증.

- 회원가입/로그인: **이메일+비밀번호** 및 **소셜 로그인 (Apple, Google)**
- iOS ↔ 웹 간 데이터 동기화 (클라우드 저장)
- 일기 데이터 **내보내기**: PDF 또는 ZIP(이미지+텍스트) 형식
- 계정 탈퇴 시 모든 데이터 삭제 처리 (GDPR/개인정보보호법 대응)

## 4. 범위 제외 (Out of Scope — MVP 미포함)

| 항목 | 제외 이유 |
|------|-----------|
| AI 자동 일기 생성 | 기술 검증 및 비용 미정, 2차 버전에서 검토 |
| AI 사진 자동 태깅/분류 | 동일 이유, 추후 기능 |
| 소셜 피드 (팔로우/좋아요/댓글) | 개인 기록 앱 정체성에 집중, 복잡도 증가 |
| Android 앱 | MVP는 iOS + 웹, Android는 2차 출시 |
| 동영상 일기 | 스토리지·처리 비용 증가, 추후 검토 |
| 다국어 지원 | 초기 타겟은 한국 사용자 |
| 구독/결제 기능 | 수익화 모델 미확정 |
| 오프라인 전용 모드 | 클라우드 동기화 구조와 충돌, 추후 고려 |

## 5. 승인 기준 (Acceptance Criteria)

### AC-01. 사진 타임라인
- [ ] 기기 갤러리 사진이 날짜별로 그룹핑되어 3초 이내 로딩됨
- [ ] 1,000장 이상 사진 환경에서 스크롤이 끊기지 않음 (60fps 유지)
- [ ] 사진 접근 거부 시 빈 상태 안내 문구와 권한 재요청 버튼이 표시됨

### AC-02. 사진 선택 → 일기 작성
- [ ] 사진 선택 후 2탭 이내에 일기 작성 화면 진입 가능
- [ ] 다른 날짜 사진 동시 선택 시 경고 메시지가 표시되고 선택이 차단됨
- [ ] 임시저장 후 앱 재진입 시 작성 중이던 내용이 복원됨

### AC-03. 일기 목록 및 상세
- [ ] 작성된 일기가 날짜 내림차순으로 목록에 노출됨
- [ ] 캘린더 뷰에서 일기 있는 날짜에 점이 표시되고, 탭 시 해당 일기로 이동함
- [ ] 삭제된 일기가 목록에서 즉시 사라짐

### AC-04. 선택적 공유
- [ ] 생성된 링크를 앱 미설치 기기의 브라우저에서 열 수 있음
- [ ] 링크 비활성화 후 해당 URL 접근 시 "만료된 링크" 안내 페이지 표시
- [ ] 공유된 일기는 수신자가 수정/삭제 불가

### AC-05. 알림
- [ ] 설정한 시각에 ±5분 이내로 푸시 알림이 발송됨
- [ ] 당일 일기 작성 완료 후에는 알림이 발송되지 않음

### AC-06. 계정 및 동기화
- [ ] iOS에서 작성한 일기가 웹 로그인 후 30초 이내 동기화됨
- [ ] 탈퇴 후 재가입 시 이전 데이터가 조회되지 않음
- [ ] PDF 내보내기 파일에 사진과 본문 텍스트가 모두 포함됨
